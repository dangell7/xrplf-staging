Executive summary

The pipeline is unusually loaded. The biggest themes are the WASM/Smart-Contract arc (XLS-100→101→102), the MPT extension family (V2 · Dynamic · Confidential · DEX · PermissionedDomain), and the Lending stack (Vault → LendingProtocol).

Risk · Spec drift

LendingProtocol shipping ahead of XLS-66

Registered Supported::Yes in develop while XLS-66 has 5+ open spec PRs (#484, #494–#497, #538). Equation (23) had a math bug a week ago. Implementation may diverge from final spec.

Block Mainnet vote until XLS-66 marked Final and impl re-diffed.

Risk · Duplicate amendment

Batch vs BatchV1_1 collision

Develop has XRPL_FEATURE(Batch, Supported::No) while PR #6446 proposes BatchV1_1 with a rename. Risk of two amendments visible to operators with no clear deprecation path.

Coordinate drop one before promoting either to Supported::Yes.

Risk · Design split

CLAMM has two competing implementations

PR #6605 (RomThpt, XLS-99 + CLAMMPosition ledger object) vs PR #5697 (LJ-XRPL, alternative design). XRPL-Standards PR #498 is a third draft without an XLS number.

Coordinate design council before either PR advances.

Risk · Validator fatigue

10+ fix amendments in the next ballot

Historically validators skip individual votes when ballots are long. Bundle related fixes; provide one-pager per amendment.

Coordinate ballot bundling with validator outreach.

Ready · Low risk

fixConstantInvariant (#7029)

Defence-in-depth invariant ensuring only mutable-declared fields are mutated. Should land before Smart Escrows — catches a class of WASM sandbox-escape that is otherwise subtle.

Sponsor promote and vote ahead of XLS-100.

Ready · Low risk

SingleAssetVault (XLS-65)

Spec is final-ish; post-merge fixes (#469, #470, #485) are operational refinements. Should activate one Mainnet round before LendingProtocol.

Sponsor publish changelog, schedule vote.

Pipeline · Long horizon

WASM smart contracts arc

XLS-100 (Smart Escrows) → XLS-101 (Smart Contracts) → XLS-102 (WASM VM). Wasmi runtime in PR #6075. Twelve-month minimum to Mainnet vote with external audit.

Coordinate spec freeze for XLS-102 first.

Pipeline · Fork-only

Transia experimental amendments

Six amendments live in Transia-RnD/rippled with no upstream PR yet: Quantum (post-quantum signing), Passkey (P-256/WebAuthn), Options (XLS-62), ImportExport, Firewall v3, SmartContract umbrella.

Coordinate upstream roadmap with Transia.

Blackhole · Resolved

fixTokenEscrowV1_1 (#7121)

Allows escrow flows for blackholed accounts (master key disabled, no regular key). Only in-flight upstream amendment that handles blackhole semantics — no other gaps in this area.

Sponsor review-ready, small surface, ship in next release.

Cross-cutting risks & process recommendations

Failure modes that span amendments and the process gates that close them.

Process · Spec drift

Code-freeze before spec-freeze

LendingProtocol in develop with Supported::Yes while XLS-66 still drifting. Pattern repeats with CLAMM and Batch.

Fix: introduce an explicit gate — an amendment cannot be merged with Supported::Yes unless the corresponding XLS is marked Final.

Process · Invariants

Invariant-per-new-object rule

Several recent amendments expand mutation surface without proportional invariant growth. PRs #7272, #7285, #7295, #6736 are post-hoc additions on already-merged work.

Fix: for every new ledger object, require a dedicated invariant in InvariantCheck.cpp verifying asset conservation across the object's lifecycle. Apply retroactively to LendingProtocol, SingleAssetVault, PermissionedDEX.

Process · Ballot

Validator fatigue at long ballots

10+ fix amendments and 4–5 features in the next ballot. Validators historically skip individual votes when ballots are dense.

Fix: bundle "uncontroversial cleanup" fixes; publish a per-amendment one-pager to validators ahead of each round.

Process · Duplicates

Batch / BatchV1_1 visibility hazard

Two amendment identifiers, one feature. Operators reading the registry will be confused.

Fix: retire one before either promotes to Supported::Yes. Document the rename in release notes.

Process · Un-flagged behaviour changes

Stop-loss / add-sentinel pattern

Transia fork branches modify protocol behaviour without registering a new feature flag. If any such pattern reaches upstream un-gated, it is a deploy-time hazard.

Fix: B4 review must confirm amendment-gating as the first action on every behaviour-changing PR. Likely-amendment PRs (#7285, #7299, #7272, #7295, #6736, #6853) should be checked first.

Process · MPT roadmap

MPT family coordination

MPTokensV2 · DynamicMPT · XLS-82 (MPT DEX) · XLS-90 (PermissionedDomain MPT) · XLS-96 (Confidential MPT) all touch the same primitive.

Fix: publish a single MPT roadmap stating which functionality lives in V2, which is independent, and the migration story for V1 issuances.

Process · WASM ordering

XLS-100 → 101 → 102 merge order

Required order: (1) fixConstantInvariant (#7029); (2) WASM runtime PR #6075; (3) XLS-102 final; (4) Smart Escrows (#6329); (5) Smart Contracts (#5809) ≥6 months later. Skipping invites Devnet correctness becoming Mainnet incidents.

Pipeline · Quick wins

PermissionedDomainInvariant & ExpiredNFTokenOfferRemoval

Registered Supported::yes across 8+ fork branches each but missing from develop. Lowest-risk, highest-readiness candidates for the next fix-bundle PR alongside Cleanup3_2_0.

Take over open the upstream PR.
